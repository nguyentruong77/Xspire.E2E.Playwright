  "Quy tắc commit code & quy trình làm việc với Git"

========================
NHIỆM VỤ DUY NHẤT
========================

Dựa trên:
- Nội dung file Markdown đã đọc
- Danh sách thay đổi được cung cấp bên dưới

➡️ Sinh ra MỘT commit message HỢP LỆ để dùng trực tiếp cho `git commit`.

========================
RÀNG BUỘC BẮT BUỘC
========================

1. Commit message PHẢI tuân thủ 100% quy ước trong file Markdown:
   - Conventional Commits
   - Tiếng Việt có dấu
   - Có tiêu đề + body
   - Body tối thiểu 2 dòng, mỗi dòng bắt đầu bằng "- "

2. CHỈ mô tả những thay đổi THỰC SỰ CÓ
   - Không suy đoán
   - Không thêm việc chưa làm

3. TUYỆT ĐỐI KHÔNG:
   - Nhắc đến quy tắc commit
   - Nhắc đến file Markdown
   - Nhắc đến prompt
   - Giải thích lý do hoặc phân tích

========================
TỰ KIỂM TRA TRƯỚC KHI TRẢ LỜI
========================

Trước khi xuất kết quả, hãy tự kiểm tra:
- [ ] Có đúng format trong file MD không?
- [ ] Có dùng tiếng Anh không?
- [ ] Có body không?
- [ ] Có từ mơ hồ không?
